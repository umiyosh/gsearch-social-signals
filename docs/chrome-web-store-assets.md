# Chrome Web Store image assets

This note records the image assets prepared for Chrome Web Store publication.

## Extension icons

The extension icon is already configured in `public/manifest.json` for both
`icons` and `action.default_icon`.

| Size    | Asset                      |
| ------- | -------------------------- |
| 16x16   | `public/icons/icon16.png`  |
| 32x32   | `public/icons/icon32.png`  |
| 48x48   | `public/icons/icon48.png`  |
| 128x128 | `public/icons/icon128.png` |

Verification:

- All four files match the required pixel dimensions.
- All four files have transparent corner pixels.

## Badge icons

Hatena and Hacker News badge icons are bundled as extension package assets.

| Badge           | Decision                        | Asset                              |
| --------------- | ------------------------------- | ---------------------------------- |
| Hatena Bookmark | Bundle in the extension package | `public/icons/hatena-bookmark.svg` |
| Hacker News     | Bundle in the extension package | `public/icons/hacker-news.svg`     |

Reasoning:

- The content script renders both icons through `chrome.runtime.getURL()`.
- `public/manifest.json` exposes the two SVG files through
  `web_accessible_resources`.
- This avoids fetching external favicon/image assets at runtime.

## Screenshots

Chrome Web Store screenshots should demonstrate the user experience and use
1280x800 or 640x400. The prepared screenshots use 1280x800 PNGs captured from
Chrome with the unpacked extension loaded from the current `dist/` directory.

| Scenario                                                | Asset                                               |
| ------------------------------------------------------- | --------------------------------------------------- |
| Google Search result with Hatena badge                  | `docs/assets/store/screenshot-hatena-badge.png`     |
| Google Search result with Hacker News badge             | `docs/assets/store/screenshot-hn-badge.png`         |
| Google Search result with Hatena and Hacker News badges | `docs/assets/store/screenshot-hatena-hn-badges.png` |
| Hatena hover overlay                                    | `docs/assets/store/screenshot-hover-overlay.png`    |

Screenshot privacy review:

- Search queries: `はてなブックマーク 使い方` and `antirez from where left`.
- The captured screenshots use actual Google Search result pages and actual
  extension-injected badges.
- Hatena and Hacker News badge images are loaded from `chrome-extension://`
  package assets, not external favicon URLs.
- The right-side Google account area is excluded from the final 1280x800 crops.
- No internal information, cookies, browser profile, or account identifier is
  visible in the final screenshot files.

## Promotional images

Chrome Web Store requires a small promotional image. The marquee promotional
image is optional, but prepared to avoid blocking later store listing work.

| Size     | Asset                                          |
| -------- | ---------------------------------------------- |
| 440x280  | `docs/assets/store/promo-small-440x280.png`    |
| 1400x560 | `docs/assets/store/promo-marquee-1400x560.png` |

The promotional images keep copy short, use the same product name as the
manifest, and visually show the Hatena/Hacker News badge pairing.

## Regeneration

Run this command from the repository root to regenerate promotional images:

```bash
node scripts/generate-store-assets.mjs
```

The script requires ImageMagick's `magick` command. Store screenshots are
captured manually from Chrome and are not regenerated by this script.
